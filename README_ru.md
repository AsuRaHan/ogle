# OGLE3D v0.3

`OGLE3D` — современный 3D‑движок/редактор, написанный на C++17 для Windows.  
Он использует Win32 для окна, OpenGL 4.x для рендеринга, EnTT‑базированную ECS‑систему, Dear ImGui для UI‑редактора, Duktape для JavaScript‑скриптов и Bullet Physics для симуляции rigid‑body.  
Проект ориентирован на чистую модульную архитектуру с отдельными менеджерами для ввода, рендеринга, физики, скриптинга и конфигурации.

## Уже реализовано

- **Win32‑окно и OpenGL 4.x**  
- **Менеджер‑базированная архитектура приложения**  
- **ECS‑мир на EnTT** с процедурно‑генерируемой тестовой сценой  
- **Гибкая система материалов** с поддержкой нескольких текстур на объект
- **Камера и ввод** через отдельный `InputManager`  
- **Сериализация мира в JSON** и сохранение/загрузка конфигурации на диск  
- **Панели редактора**: вид мира, иерархия, инспектор, браузер контента  
- **JavaScript‑runtime** через Duktape с C++ биндингами dukglue  
- **Симуляция физики** с поддержкой статических, динамических и кинематических твёрдых тел (Box, Sphere, Capsule)
- **Структуры данных для скелетной анимации** (клипы, треки, ключевые кадры)

## Текущее состояние

Проект активно развивается. В нём уже можно собирать небольшую сцену прямо в редакторе, просматривать ассеты, назначать текстуры, выбирать объекты во viewport, управлять симуляцией и сохранять/загружать состояние мира. Многие системы остаются простыми, но кодовая база уже пригодна для экспериментов и дальнейшего развития инструментов.

## Что уже реализовано

### Архитектура приложения
- `App` теперь координирует работу, а не является god‑объектом  
- Логика разделена по отдельным менеджерам  
- Пакеты `config`, `world`, `rendering`, `input`, `editor`, `scripting` и `physics` вынесены в отдельные подсистемы  

### Ввод
- Старые input‑классы адаптированы под текущий проект  
- `InputManager` получает Win32‑сообщения и обновляет `InputController`  
- `InputActionsManager` настраивает действия и управление камерой  
- Рабочие команды: `W A S D`, `Q / E`, `Shift`, правый клик мыши для обзора, `Esc` для закрытия приложения  
- Исправлены проблемы с отрицательными значениями осей и потерей фокуса  

### Рендеринг и окно
- Инициализация OpenGL и рендер кадра вынесены в `RenderManager`  
- Тайминг кадра в `TimeManager`  
- Логика камеры в `CameraManager`  
- Исправлены проблемы DPI/resize/viewports  
- Объекты ECS‑мира рендерятся напрямую из данных мира  
- Гибкая система материалов с поддержкой нескольких текстур на объект.
- Выбранный объект подсвечивается во viewport; флаги `enabled` и `visible` реально влияют на рендер  

### ECS‑мир
- `WorldManager` владеет активным миром  
- `World` построен на EnTT  
- `WorldObject` — лёгкая обёртка над entity  
- Используемые компоненты:  
  - `WorldObjectComponent`: Базовое состояние (включен, видим).
  - `NameComponent`: Имя сущности.
  - `TransformComponent`: Позиция, вращение, масштаб.
  - `ModelComponent`: Ссылка на ресурс 3D-модели.
  - `PrimitiveComponent`: Описывает, как была создана геометрия (например, куб, сфера, из файла).
  - `MaterialComponent`: Определяет внешний вид объекта, может содержать несколько текстур.
  - `ShaderComponent`: Указывает, какую шейдерную программу использовать для рендеринга.
  - `LightComponent`: Определяет источник света (направленный или точечный).
  - `PhysicsBodyComponent`: Описывает физическое тело для симуляции (тип, форма, масса и т.д.).
  - `ScriptComponent`: Привязывает JavaScript-файл к сущности.
  - `AnimationComponent`: Управляет состоянием анимации (проигрывание, зацикливание, текущее время).
  - `SkeletonComponent`: Хранит данные скелета для анимированных моделей (в разработке).
- Существует тестовый мир по умолчанию  
- Механизм сохранения/загрузки мира в JSON реализован  
- Процедурная геометрия сериализуется и сохраняется  

### Процедурные объекты
- Кубы создаются без загрузки файлов  
- Тестовый мир строится процедурно  
- Мир можно расширять из C++ и скриптов  
- Процедурные кубы могут использовать текстуры  

### Ассеты и модели
- Модели загружаются через Assimp  
- Путь к diffuse‑текстуре может импортироваться из материала модели  
- Существует базовая система материалов и кэш текстур  
- В редакторе добавлен контент‑браузер с корнем в настроенной папке ассетов  

### Скрипты
- Duktape интегрирован  
- `ScriptManager` загружает `.js`‑файлы  
- Поддерживаются `onStart()` и `onUpdate(dt)`  
- Скрипты могут создавать и изменять world‑объекты, назначать текстуры  
- Расширенный JavaScript API для создания света, обработки ввода и реагирования на физические столкновения (см. `docs/js.md`)

Пример скрипта:

```js
var cubeId = null;

function onStart() {
    cubeId = ogle.world.createCube("ScriptedCube", [0, 1, 0], [1, 1, 1]);
    ogle.log("Куб создан с ID: " + cubeId);
}

function onUpdate(dt) {
    if (cubeId !== null && ogle.entity.exists(cubeId)) {
        var rot = ogle.entity.getRotation(cubeId);
        rot[1] += 45.0 * dt; // Вращение вокруг оси Y
        ogle.entity.setRotation(cubeId, rot);
    }
}
```
### UI и редактор
- Dear ImGui интегрирован  
- Старые debug/demo‑окна скрыты по умолчанию  
- `Editor` вынесен как отдельный слой и может включаться/выключаться  
- Реализовано выделение объектов мышью во viewport  
- Редактор использует отдельные окна: `World`, `Hierarchy`, `Inspector`, `Content Browser`  
- `Hierarchy` показывает объекты сцены и поддерживает быстрые действия добавления/удаления  
- `Inspector` позволяет редактировать: трансформ, состояние объекта, назначение текстуры, простые настройки физики  
- `Content Browser` работает от корня, заданного в конфиге, и поддерживает drag‑and‑drop: модель → окно `World` (создание сущности), текстура → `Inspector` (назначение)  

### Физика
- Bullet Physics интегрирован  
- `PhysicsManager` владеет Bullet‑миром  
- Поддерживаются статические, динамические и кинематические твёрдые тела с формами коллайдеров Box, Sphere и Capsule.
- Тела Bullet синхронизируют трансформ обратно в ECS  
- Тестовая сцена создаёт статические и динамические объекты  
- События столкновений отправляются и могут быть обработаны в скриптах.

### Конфиг и файловая система
- Добавлен `FileSystem` для работы с путями и текстовыми файлами  
- Созданы `AppConfig` и `ConfigManager`  
- Конфиг загружается с диска либо создаётся с дефолтами  
- Настраиваются параметры окна, состояние редактора, путь к миру, ассетам и скриптам запуска
