## ТЗ для ИИ-агента: Создание системы Shader & ShaderManager
Контекст: В проекте есть старый ShaderManager на строковых мапах. Нужно заменить его на современную систему, где шейдер — это самостоятельный объект с кэшированием юниформов.
## 1. Создать класс OGLE::Shader

* Инкапсуляция: Класс должен владеть GLuint m_programId.
* RAII: Запретить копирование, реализовать удаление программы в деструкторе.
* Методы SetUniform: Реализовать перегрузки SetUniform для int, float, vec2/3/4, mat4 (использовать glm).
* Кэширование:
* Внутри иметь std::unordered_map<std::string, GLint> m_uniformCache.
   * Метод GetUniformLocation(name) должен сначала проверять мап. Если локации нет — вызывать glGetUniformLocation и сохранять её в мап.
* Интерфейс: Bind(), Unbind(), IsValid().

## 2. Создать/Обновить OGLE::ShaderManager (Singleton)

* Хранилище: Вместо std::map<string, GLuint> использовать std::map<std::string, std::shared_ptr<Shader>> m_shaders.
* Загрузка:
* Load(name, vsPath, fsPath): Читает файлы через FileSystem, компилирует и возвращает std::shared_ptr<Shader>.
   * Если шейдер с таким name уже есть — возвращать его.
* Hot Reload:
* Метод Reload(name): Должен перечитать файлы с диска.
   * Важно: Сначала скомпилировать новую программу в локальную переменную. Если успех — обновить m_programId у существующего объекта Shader (чтобы все Material, которые его держат, обновились автоматически). Если ошибка — вывести лог и оставить старую версию.

## 3. Интеграция с OGLE::Material

* Поле: Заменить std::string m_shaderProgramName на std::shared_ptr<Shader> m_shader.
* Метод Bind: Переписать Material::Bind, чтобы он использовал m_shader->SetUniform(...) вместо прямых вызовов OpenGL и ручного поиска локаций.

## 4. Требования к реализации

* Использовать имеющийся в проекте Logger.
* Использовать glm для математики.
* Код должен быть в неймспейсе ogle как и везде в проекте.
* .\build.bat Release проверить сборку. исправить ишибки если будут.
